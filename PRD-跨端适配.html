1背景与目标

问题陈述

当前「小丑牌」游戏仅有网页版（Vue 3 + Vite），无法以原生 App 形式分发到 Android / iOS / macOS / Windows 平台，用户需要打开浏览器才能游玩，体验割裂，且无法上架应用商店。

目标平台

Android iOS macOS Windows 网页版（保持现状）

产品目标

  • 用 Tauri 2 把现有 Vue 3 网页游戏打包为 5 端原生/混合 App，一套代码多端分发
  • 移动端（Android / iOS）强制横屏，保证游戏可玩性
  • UI 层面在移动端小屏幕（横屏高度 ≤ 500px）下保持完整可操作，不裁掉任何功能
  • 网页版现有布局、交互、体验完全不受影响
  • 本轮不改动任何游戏逻辑（composables 层代码冻结）

成功指标

指标定义目标值
跨端构建通过率5 个平台均能成功构建且启动5/5
网页版回归现有网页版功能无任何 UI 回归0 个回归
移动端可操作性手机横屏下所有按钮可点击、牌可见100%
强制横屏生效Android / iOS 竖屏时弹出旋转提示或系统锁定横屏两端均生效
2目标用户

用户画像

主要用户：课程学员 / 开发者，在手机 / 平板 / 电脑上体验游戏，关注跨端一致性

次要用户：移动端休闲玩家，习惯横屏操作卡牌游戏

用户旅程（移动端）

  1. 用户在手机上安装小丑牌 App（Android APK 或 iOS IPA）
  2. 竖屏打开时，看到「请横屏游玩」提示，将手机横过来
  3. 进入游戏开始界面，触摸操作正常，UI 完整可见
  4. 完成一局游戏，体验与网页版功能一致
3需求详述

功能清单（按优先级）

  • P0Tauri 2 集成：在现有 Vue 3 + Vite 项目中引入 @tauri-apps/cli@^2 与 @tauri-apps/api@^2，完成 src-tauri 初始化，保证 npm run dev 与 Tauri dev 模式并存不冲突
  • P0移动端强制横屏：Android 通过 AndroidManifest.xml 设置 screenOrientation="landscape"；iOS 通过 Info.plist 限定 UISupportedInterfaceOrientations 为 landscape。前端兜底：竖屏时显示「请横屏游玩」全屏遮罩
  • P0移动端横屏小屏 UI 适配：手机横屏场景（高度 ≤ 500px），SideBar 宽度收窄、字号缩小；HandArea 卡牌缩小（100×145 → 60×88px）、操作按钮紧凑；PlayArea 出牌区卡牌与牌堆等比缩放
  • P0Viewport meta 适配：禁止用户双指缩放（user-scalable=no），防止移动端误触放大破坏布局
  • P1桌面端窗口配置：macOS / Windows 设置初始窗口尺寸 1280×800，最小尺寸 900×560，防止窗口过小导致布局错乱
  • P1网页版零回归：所有移动端适配样式通过媒体查询隔离（@media (max-height: 500px) and (orientation: landscape)），桌面浏览器不触发
  • P2触摸友好性：移动端按钮点击区域 ≥ 36px，防止误触

User Story

Story 1 — 移动端强制横屏

作为 手机玩家，我希望 打开游戏时自动锁定横屏，以便 无需手动旋转手机即可看到完整界面。

Given 用户以竖屏方向打开 App

When App 加载完成

Then 系统弹出横屏锁定（或显示「请横屏游玩」遮罩），游戏主界面不在竖屏下渲染

Story 2 — 移动端横屏可玩

作为 手机玩家（横屏），我希望 所有手牌、按钮、侧边栏信息都在屏幕内可见且可点击，以便 正常游玩而不需要滚动。

Given 用户使用主流手机横屏（高度约 360–430px）

When 进入游戏 playing 阶段

Then 侧边栏、出牌区、手牌区、操作按钮全部无遮挡可见，无需滚动

Story 3 — 网页版零回归

作为 网页版玩家，我希望 跨端适配改动后网页版界面与之前完全一致，以便 不受任何影响。

Given 用户在桌面浏览器访问网页版（宽度 > 900px）

When 游戏正常运行

Then 布局、字号、动画与改动前 100% 一致，无任何视觉差异

4交互与边界

各端布局策略

平台屏幕特征布局策略
网页版 / 桌面宽 ≥ 900px原有布局，完全不变
macOS / Windows App窗口 1280×800，可缩放原有布局 + 最小窗口限制
手机横屏（Android/iOS）高度 360–500px紧凑模式：侧栏收窄、卡牌缩小、按钮压缩
手机竖屏宽度 < 高度全屏「请横屏游玩」遮罩，不渲染游戏
平板横屏高度 ≥ 600px与桌面端一致，走原有布局

媒体查询断点

断点触发条件作用
移动端横屏紧凑模式max-height: 500px + orientation: landscapeSideBar / HandArea / PlayArea / PlayingCard 缩放
移动端竖屏遮罩max-width: 768px + orientation: portrait显示旋转提示层

不在本轮范围内（Out of Scope）

  • 任何游戏逻辑修改（出牌、计分、AI、Joker 效果等）
  • 触摸手势扩展（滑动出牌、长按选牌等）
  • 实际上架 App Store / Google Play
  • 平板竖屏布局优化
  • 推送通知、后台运行等原生能力
  • 性能优化、包体积压缩

6风险与假设
风险概率影响应对策略
Tauri 2 iOS/Android 仍为 Beta，API 可能变动 中 高 锁定 Tauri 版本；本轮仅做 UI 适配，不调用原生 API，降低风险面
部分手机横屏高度 < 360px，紧凑布局仍溢出 低 中 以 iPhone SE 横屏（高度 375px）为最小基准测试；极端情况允许内容区滚动
iOS WKWebView 字体渲染与桌面差异导致卡牌显示异常 中 低 字体使用系统 fallback（PingFang SC），Google Fonts 仅作增强
Android NDK 环境未安装导致构建失败 中 中 构建前检查 NDK 环境；macOS/Windows 构建不依赖 NDK

关键假设

  • 游戏逻辑（composables）本轮完全冻结，不存在因适配引起的逻辑 bug
  • 目标手机横屏高度 ≥ 360px（覆盖市面 95% 以上机型）
  • 网页版用户使用桌面浏览器，不会触发移动端媒体查询断点
  • Tauri 生成的 WebView 支持标准 CSS media query
